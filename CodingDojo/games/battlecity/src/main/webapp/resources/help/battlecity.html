Battlecity codenjoy — how to play?

В чем суть игры?

Надо написать своего бота для танчика, который обыграет других ботов по очкам. Вся игра при этом проходит на одном поле. Танчик может передвигаться по свободным ячейкам во все четыре стороны. Так же танчик может выстрелить снарядом, который взорвется при попадании в препятствие. Снаряд движется быстрее танка в два раза.

За убийство врагов боту игрока начисляются очки. За танк-бота +25 очков. За танк другого игрока +50 помноженное ан количество уничтоженных танков с момента последнего уничтожения твоего танка - так что выгодно оставаться живым как можно дольше. Очки суммируются. Побеждает игрок с большим числом очков (до условленного времени). Мертвый танк тут же появляется в random месте на поле.

Итак, игрок регистрируется на сервере, указывая свой email и имя

Далее необходимо подключиться из кода к серверу через вебсокеты. Это Maven проект и подойдет он для игры на JVM языках. Так же есть JavaScript клиент. Как его запустить смотри в корне проекта в файле README.md

Идея подключения одинакова для всех клиентов. Копируешь url из из адресной строчки браузера скрина с полем (сразу после регистрации/логина) и используешь его в своем клиенте для подключения. Например у меня этот линк такой - ищи эту строчку в своем клиенте и вставь туда свой url

Игра пошаговая, каждую секунду сервер посылает твоему клиенту (боту) состояние обновленного поля на текущий момент и ожидает ответа команды танку. За следующую секунду игрок должен успеть дать команду танку. Если не успел — танк остается на месте.

Команд несколько: UP, DOWN, LEFT, RIGHT – приводят к развороту и передвижению танка в заданном направлении на 1 клетку; ACT - выстрел снаряда. Команды движения можно комбинировать с командами выстрела, разделяя их через запятую – это значит что за один такт игры будет и выстрел и движение (LEFT, ACT) или (ACT, LEFT), что аналогично

Первая задача – запустить websocket клиента, который подключится к серверу так чтобы танк слушался твои команды. Следом стоит вести осмысленную игру и победить на финале.

Игра с открытым исходным кодом. Для реализации своей игры, исправления ошибок в текущей и внесения других правок необходимо для начала форкнуть проект. В корне репозитория есть описание в файле Readme.md - там описано, что делать дальше.
